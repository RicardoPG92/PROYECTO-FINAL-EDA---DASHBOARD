TÍTULO DEL PROYECTO 
  ANÁLISIS EXPLORATORIO Y PRESENTACIÓN MEDIANTE PANEL DE DATOS DE UNA SERIE DE DATOS DE VENTAS DE SUPERMERCADOS
   
   DESCRIPCIÓN DE LO **TRABAJOS PREVIOS** REALIZADOS:
      ESTE DATASET CONTIENE UN ARCHIVO CSV QUE DEBEMOS IMPORTAR Y TRATAR EN UN FICHERO EXCEL PARA DARLE UN FORMATO INTELIGIBLE Y EN QUE PODAMOS TRABAJAR. 
      LA IMPORTACIÓN A EXCEL DE LOS DATOS LA HACEMOS MEDIANTE LA FUNCIONALIDAD DE OBTENER DATOS DESDE CSV EN LA PESTAÑA DATOS. cARGAMOS LOS DATOS DEJANDO QUE EL SISTEMA DETECTE EL TIPO DE DATOS EN FUNCIÓN DE LOS 200 PRIMEROS REGISTROS. 
      LA BBDD RESULTANTE CONTIENE 15 COLUMNAS Y 5000 FILAS CON DIFERENTES REGISTROS. 
      ESTE DATASET CONTIENE REGISTROS DE TRANSACCIONES DE VENTAS DE UNA EMPRESA DEDICADA APROVISIONAMIENTO DE PRODUCTOS EN CATEGROÍAS COMO LA ELECTRÓNICA, MOBILIARIO O ACCESORIOS DEL HOGAR. LA EMPRESA OPERA EN VARIAS REGIONES: NORTE AMÉRICA, SUDAMÉRICA, EUROPA Y ASIA. CADA REGISTRO CONTIENE DIFERENTES DETALLES DEL PROCESO DE VENTA INCLUYENDO NÚMERO DE TRANSACCIÓN, SEGMENTO DE CLIENTES, REPRESENTANTE DE VENTAS, EQUIPO, CATEGORÍA DE PRODUCTO, ETC…
      EN UNA PRIMERA EXPLORACIÓN ANALIZAMOS LOS DIFERENTES DATOS DE CADA COLUMNA Y COMENZAMOS CON EL ANÁLISIS EXPLORATORIO DEL DATO:

  **EDA - ANÁLISIS EXPLORATORIO DEL DATO**
    EN UN PRIMER ANÁLISIS DE ESTE CONJUNTO DE DATOS VEMOS QUE HAY COLUMNAS CON DIFERENTES TIPOS DE FORMATOS Y CLASES DE DATO. DATOS NUMÉRICOS, DATOS TEMPORALES, DATOS CATEGÓRICOS, DATOS GEOGRÁFICOS, ETC...
    LO PRIMERO QUE DEBEMOS HACER ES VER SI EL FORMATO ES EL CORRECTO PARA LOS DATOS INCLUIDOS EN CADA COLUMNA.
    UNA VEZ REALIZADA ESTA COMPROBACIÓN VAMOS A REVISAR SI HAY CELDAS VACIAS O SI EXISTE ALGÚN REGISTRO DUPLICADO. 
      EN UNA COLUMNA NUEVA DENOMINADA "COMP DUPLICADOS" BUSCAMOS CADA REGISTRO DE LA COLUMNA "ORDER_ID" EN EL RANGO QUE NOS QUEDARÍA HASTA EL FINAL DE LA COLUMNA CON LA FX SI CONCATENADA CON LA FORMULA BUSCAR V. CONVERTIMOS EL RESULTADO EN UNA VARIABLE BOOLEANA DE 1 Y 0 COMO NO ENCUENTRA NINGUNA HEMOS COMPROBADO QUE NO EXISTEN REGISTROS DUPLICADOS. 
      PARA REVISAR SI HAY CELDAS VACÍAS Y RELLENARLAS SI FUERA NECESARIO, EN OTRA PESTAÑA "COMPROBACIONES" NOS LLEVAMOS LOS ENCABEZADOS DE COLUMNA  Y EN CADA UNA APLICAMOS LA FORMULA CONTAR.BLANCO PARA VER SI HAY REGISTROS VACÍOS. EN ESTE CASO NO LOS HAY.
    
    DESPUÉS DE HACER ESTAS COMPROBACIONES VAMOS A ESTUDIAR MÁS EN PROFUNDIDAD LOS DATOS Y HACER LAS TRANSFORMACIONES QUE QUERAMOS. EN EL CASO DE LOS DATOS DE FECHA HE INCLUIDO A RAÍZ DEL DATO ORIGINAL DIFERENTES VARIABLES  ( DÍA, MES, AÑO, DIA SEM). PARA ELLO ADEMÁS DEL USO DE DIFERENTES FORMULAS HE UTILIZADO DIFERENTES LISTAS QUE SE INCLUYEN EN LA PESTAÑA "MES DÍA"
    ADEMÁS DE LA TRANSFORMACIÓN DE LAS FECHAS VAMOS A SACAR UN KPI IMPORTANTE QUE ES EL COSTE MEDIO LAS UNIDADES VENDIDAS

  UNA VEZ REALIZADO EL ANÁLISIS DE LOS DATOS VAMOS A PROCEDER CON SU **ANÁLISIS** PARA ELLO VAMOS A REALIZAR DIFERENTES TABLAS DINÁMICAS EN UNA NUEVA PESTAÑA.

  SACAMOS LOS BIG NUMBERS Y HACEMOS UN ANÁLISIS POR EQUIPO Y CATEGORÍA. ESTO PUEDE DEBERSE A QUE SE VENDIERON EN ESTA LINEA DE NEGOCIO ARTÍCULOS DE MÁS VALOR O QUE EL PRECIO AUN SIENDO LOS MISMOS ARTÍCULOS ES MÁS ELEVADO.
